你是一名高级全栈工程师。

生成一个完整的跨平台桌面应用程序，名为**OneStepOpenClaw**。

该软件的目标是让非技术人员能够轻松地安装、配置和管理 OpenClaw。

该应用程序必须在**Windows 和 macOS**上运行。

使用 Electron 作为桌面框架。

---

# 技术栈

使用以下技术栈：

- Electron（桌面框架）
- Node.js（系统命令与后端逻辑）
- React + Vite（前端 UI）
- TailwindCSS（UI 样式）
- Shadcn UI 组件（现代 UI 组件）
- pnpm（包管理器）

尽可能在所有地方使用 TypeScript。

---

# 核心产品理念

OneStepOpenClaw 是一个图形界面工具，允许用户：

1. 自动安装 OpenClaw
2. 配置 API 密钥
3. 启动/停止 OpenClaw 网关
4. 查看日志
5. 切换模型
6. 管理配置文件

该软件应消除终端使用的需求。

---

目标用户是：

* 不熟悉命令行的初学者

# 页面详情

## 仪表盘

显示系统状态。

卡片：

OpenClaw 状态
节点版本
网关状态
当前已选择模型

使用卡片组件。

---

## 安装页面

特性：

检测 OpenClaw 是否已安装。

如果未安装：

显示按钮：

安装 OpenClaw

安装逻辑：

使用 Node child_process 运行系统命令。

示例命令：

pnpm install -g openclaw

显示进度条。

显示实时安装输出。

安装完成后显示成功/失败状态。

---

## 配置页面

允许用户配置：

API 密钥
基本 URL
默认模型

字段：

API 密钥输入
基础 URL 输入
模型下拉菜单

保存配置到：

.env 文件

示例字段：

ANTHROPIC_AUTH_TOKEN
ANTHROPIC_BASE_URL
ANTHROPIC_DEFAULT_MODEL

---

## 网关控制页面

按钮：

启动网关
停止网关
重启网关

当用户点击开始：

运行命令：

openclaw gateway start

显示运行状态指示器。

---

## 日志页面

在终端风格的查看器中显示日志。

功能：

自动滚动
清除日志按钮

---

## 设置页面

允许用户配置：

启动时自动启动
默认模型
主题（暗色 / 亮色）

# 系统集成

使用 Electron 主进程运行系统命令。

child_process.exec

所有终端命令应在 Node 后端执行。

前端应使用 Electron IPC 进行通信。
